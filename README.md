# Relo Dojo

A mobile app for learning English with short daily grammar exercises, with examples drawn from the
learner's own field. Exercises are generated by an LLM through a pluggable provider (`LLM_PROVIDER`):
self-hosted **Ollama** in local dev, an **API provider** (Anthropic / OpenAI; Gemini under
evaluation) in production. Interactive answers are graded deterministically on the backend; free-text
is graded by the LLM. Mobile app: Expo (React Native). Backend: Python + FastAPI.

> Full project handoff lives in the Obsidian vault (not in this repo).

## Structure

```
backend/   FastAPI service (Python)
mobile/    Expo app (React Native + TypeScript, expo-router)
```

## Status

- **Phase 0–2 — done:** skeleton + connectivity, chat over Ollama, learning core.
- **Phase 2.5 — done:** interactive tap-based exercises (multiple-choice, build-the-sentence with
  RU→EN translation, match-pairs, tap-the-error). Interactive answers are graded server-side without
  the LLM; the correct answer is sealed in a token and never sent to the client up front. Free-text
  answers are graded by the LLM (`POST /check-answer`).
- **Phase 3 — done:** local progress & gamification (XP, levels, daily streak + best correct run,
  per-topic stats, achievements) persisted via AsyncStorage. App has Practice / Progress tabs.
- **Next → Phase 4:** accounts (PostgreSQL, auth, progress sync). See the vault backlog.

## API

| Endpoint | Purpose |
|---|---|
| `GET /health` | liveness |
| `POST /exercise` | generate an exercise (interactive types carry a sealed `token`) |
| `POST /story` | generate a themed mini-story (a sequence of linked exercises) |
| `POST /check` | deterministic grade of an interactive answer (no LLM) |
| `POST /check-answer` | LLM grade of a free-text answer + explanation |
| `POST /explain` | on-demand LLM teaching note for an interactive miss |

### Ollama (local dev)

```bash
ollama serve                  # start the server (keep running)
ollama pull gemma3:4b         # download a model (set the same name in backend/.env)
```

Model name is read from `OLLAMA_MODEL` in `backend/.env` (default `gemma3:4b`). See `.env.example`.
`CHECK_SECRET` (optional) is a Fernet key that seals exercise tokens across restarts; if unset an
ephemeral key is generated per run (fine for dev).

### Accounts DB (Phase 4 — required env)

The backend now needs PostgreSQL and a JWT secret. Required vars in `backend/.env` (no defaults —
the server refuses to start without them):

```bash
DATABASE_URL=postgresql+asyncpg://USER:PASSWORD@localhost/relo_dojo
JWT_SECRET=<random string>          # e.g. python -c "import secrets; print(secrets.token_urlsafe(48))"
ALLOWED_ORIGINS=http://localhost:8081   # comma-separated CORS origins (no wildcard)
# JWT_EXPIRE_MIN=10080               # optional, default 7 days
```

One-time setup, then apply migrations:

```bash
createdb relo_dojo               # provision Postgres + the database
cd backend && source .venv/bin/activate
alembic upgrade head                # creates the users + progress tables
```

### Run the backend

```bash
cd backend
python3 -m venv .venv          # first time only
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

Health check: `curl http://localhost:8000/health` → `{"status":"ok"}`
Exercise check: `curl -X POST localhost:8000/exercise -H 'Content-Type: application/json' -d '{}'`

### Run the mobile app

```bash
cd mobile
npm install
npx expo start
```

Open **Expo Go** (SDK 54) on a phone connected to the **same Wi-Fi** and load the project.

> The app **auto-detects** the backend host from the Expo dev host (no hardcoded IP). Override with
> `EXPO_PUBLIC_API_URL` if needed (e.g. tunnel mode).

### Done when

With Ollama running and a model pulled: open the app → get a tap-based exercise → answer it → get an
instant verdict (and an Explain note on a miss). Progress persists across restarts on the Progress tab.
